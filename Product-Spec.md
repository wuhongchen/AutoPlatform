# 产品需求文档 (Product Specification)

## 项目概述

**项目名称**：AutoPlatform

**一句话描述**：一个面向公众号运营的多账户内容后台，覆盖“采集灵感 -> AI 改写 -> 模板排版 -> 微信草稿发布”的完整流程。

**目标**：
1. 让运营人员在一个后台完成内容生产全流程，减少在多工具间切换。
2. 保证“账户维度一致”，避免统计、列表、发布上下文错位。
3. 将 AI 改写能力产品化为可配置、可追踪、可回放的标准能力。

## 目标用户

**核心用户群体**：公众号个人运营者、小型内容团队、账号矩阵运营负责人。

**用户特征**：
- 年龄段：23-45 岁
- 技术水平：中低到中等（可理解后台操作，不依赖开发能力）
- 使用场景：日更/周更内容生产、热点跟进改写、多账号协同发布
- 痛点：灵感收集分散、改写质量波动、跨账号数据混乱、发布链路不可追踪

## 核心功能

### 功能 1：多账户上下文与全局切换

**功能描述**：在后台顶部统一切换账号，驱动统计、灵感、文章与改写参考都使用同一账户上下文。

**输入**：
- `account_id`：字符串，示例 `acc_flow`
- 页面上下文：当前路由（`/dashboard`、`/articles` 等）

**输出**：
- 当前选中账户状态：持久化到 `localStorage`
- 页面数据刷新：按账户过滤后的统计与列表

**业务规则**：
- 首次进入页面时默认选中第一个可用账户。
- 若本地缓存账户已被删除，自动回退为第一个可用账户。

**异常处理**：
- 账户列表为空：允许查看空态，不阻塞后台访问。
- 账户不存在：前端自动回退并提示。

**优先级**：高

**状态**：已开发

---

### 功能 2：灵感采集与采纳建文

**功能描述**：输入 URL 采集文章内容，完成 AI 评分后进入待决策，用户采纳后自动创建待改写文章。

**输入**：
- `url`：字符串，示例 `https://mp.weixin.qq.com/s/xxx`
- `account_id`：字符串，示例 `acc_a`

**输出**：
- 灵感记录：包含标题、作者、正文、图片、AI 评分与方向
- 采纳结果：创建 `article`，并在灵感记录写入 `article_id`

**业务规则**：
- 采集后状态为“待分析”，AI 分析完成后进入“待决策”。
- 采纳动作只允许对存在的灵感记录执行。
- 灵感记录必须绑定账户 ID。

**异常处理**：
- 采集失败：返回错误并阻止入库。
- AI 评分失败：状态仍进入“待决策”，并落 `error_message`。

**优先级**：高

**状态**：已开发

---

### 功能 3：AI 改写（风格 + 参考素材 + 额外指令）

**功能描述**：对文章执行 AI 改写，支持风格预设、参考灵感、额外指令，输出可发布 HTML。

**输入**：
- `article_id`：字符串，示例 `6b8c...`
- `style`：字符串，示例 `tech_expert`
- `inspiration_ids`：数组，最多 5 个
- `instructions`：字符串，示例“增加数据支撑，语气更口语化”

**输出**：
- `rewritten_html`：HTML 字符串
- `rewrite_style`：本次风格 ID
- `rewrite_references`：本次引用参考集合
- `metadata.reference_count`：参考条数

**业务规则**：
- 改写前文章状态必须可进入改写（`pending`/`failed`）。
- 手动参考最多 5 篇。
- 改写成功状态转 `rewritten`，失败转 `failed` 并写错误。

**异常处理**：
- 文章不存在：返回 4xx 错误。
- 模型调用超时/失败：写 `error_message`，支持后续重试。

**优先级**：高

**状态**：已开发

---

### 功能 3.5：后台写稿与文章编辑

**功能描述**：允许运营人员直接在后台新建文章、编辑正文内容，并决定保存为草稿还是可直接发布的成稿。

**输入**：
- `source_title`：文章标题
- `source_author`：文章作者
- `content`：正文，支持纯文本或 HTML
- `publish_ready`：布尔值，是否保存为可发布成稿

**输出**：
- 新建或更新后的文章记录
- 状态为 `pending`（草稿）或 `rewritten`（可直接发布）

**业务规则**：
- 标题与正文不能为空。
- 纯文本会自动转为基础 HTML 段落结构。
- 已发布文章默认不在后台再次编辑。

**异常处理**：
- 内容为空返回失败。
- 发布中的文章不允许编辑。

**优先级**：高

**状态**：已开发

---

### 功能 4：模板排版与微信草稿发布

**功能描述**：将改写结果按模板渲染，并创建公众号草稿。

**输入**：
- `article_id`：字符串
- `template`：字符串，支持 `default/minimal/tech/business`
- 账户微信配置：`wechat_appid`、`wechat_secret`

**输出**：
- 发布结果：`wechat_draft_id`
- 文章状态：`published`
- 发布元数据：`published_at`、模板信息

**业务规则**：
- 未配置微信凭证时禁止发布。
- 发布前文章通常应为 `rewritten`（当前实现可直接调用，业务侧建议收紧）。
- 模板不存在时回退 `default`（服务层容错）。

**异常处理**：
- 微信接口失败：状态转 `failed`，记录错误。
- 草稿创建失败：返回错误，不写成功状态。

**优先级**：高

**状态**：已开发

---

### 功能 4.5：账户级广告位配置

**功能描述**：在账户维度配置固定头部和底部广告位内容，并在发布到微信公众号时自动插入。

**输入**：
- `ad_header_html`：头部广告位 HTML
- `ad_footer_html`：底部广告位 HTML

**输出**：
- 发布后的公众号草稿正文包含头部 / 底部广告位内容

**业务规则**：
- 广告位配置跟随账户，不跟随单篇文章。
- 发布时未配置广告位则跳过，不影响正文。

**异常处理**：
- 广告位内容为空时不报错。

**优先级**：高

**状态**：已开发

---

### 功能 5：风格预设管理（内置 + 自定义）

**功能描述**：提供 8 个内置风格和自定义风格管理，支持启用/禁用、编辑、删除（自定义）。

**输入**：
- 风格基础字段：`name`、`description`、`system_prompt`
- 生成参数：`tone`、`style`、`temperature`、`max_tokens`、`params`

**输出**：
- 风格列表
- 新建/更新/删除结果
- 启用状态更新结果

**业务规则**：
- 内置风格不可编辑/删除。
- 仅启用风格用于改写选择。
- 列表排序优先内置风格与高使用频次。

**异常处理**：
- 非法参数（如 JSON 格式错误）返回失败。
- 尝试编辑内置风格返回错误。

**优先级**：高

**状态**：已开发（使用次数回写待完善）

---

### 功能 6：账户广告位配置（账号维度）

**功能描述**：为指定账号配置头部和底部广告位，在文章发布到微信公众号时自动插入统一的图文广告内容。

**输入**：
- `account_id`：字符串
- `ad_slots.header` / `ad_slots.footer`：对象，包含标题、说明、图片、按钮文案、跳转链接

**输出**：
- `ad_header_html` / `ad_footer_html`
- 广告位预览结果

**业务规则**：
- 广告位配置独立于账户基础信息编辑。
- 运营人员无需直接编写 HTML，可通过标题、说明、图片和按钮配置生成。
- 发布时按账号自动注入头部 / 底部广告位；未配置则跳过。

**异常处理**：
- 图片上传失败：保留当前表单并提示重试。
- 广告位为空：允许保存为空配置，不阻塞发布。

**优先级**：中

**状态**：已开发

---

### 功能 7：后台运营视图（概览/文章/灵感）

**功能描述**：提供后台页面查看统计、筛选文章状态、搜索灵感、查看详情并执行关键动作。

**输入**：
- 查询条件：`account_id`、页面筛选（状态、关键词）
- 操作输入：采纳、改写、发布

**输出**：
- 统计卡片数据
- 文章列表/灵感列表
- 操作反馈（成功/失败）

**业务规则**：
- 文章页支持本地状态筛选：全部/待改写/已改写/已发布。
- 灵感页支持关键词搜索（前端过滤）。
- 改写页与当前账户联动加载参考灵感。

**异常处理**：
- API 失败统一提示可读错误。
- 空数据场景使用空态展示。

**优先级**：高

**状态**：已开发

---

### 功能 8：CLI 与脚本化运维入口

**功能描述**：通过 CLI、`run.sh`、`Makefile` 支持本地运行、测试和批处理，便于研发/运营自动化。

**输入**：
- CLI 参数：如 `--account`、`--style`、`--template`
- 脚本命令：`./run.sh backend|test|smoke`、`make test|smoke`

**输出**：
- 命令执行结果与日志
- 冒烟测试/全量测试通过状态

**业务规则**：
- `smoke` 仅覆盖关键链路（健康、账户与统计、灵感到发布主流程）。
- `test` 覆盖全量 e2e 用例。

**异常处理**：
- 缺依赖自动安装并提示。
- 参数缺失时脚本明确报错。

**优先级**：中

**状态**：已开发

## 功能优先级

### 高优先级（必须有）
1. 多账户上下文与全局切换 - 这是全后台数据正确性的前提。
2. 灵感采集与采纳建文 - 这是内容生产链路起点。
3. AI 改写（风格 + 参考灵感） - 这是平台核心价值能力。
4. 模板排版与微信草稿发布 - 这是最终交付出口。
5. 后台运营视图（概览/文章/灵感） - 决定日常可操作性。

### 中优先级（应该有）
1. 账户广告位配置 - 保证统一品牌露出与转化入口。
2. 风格预设管理增强（含使用次数回写） - 提升可运营性。
3. CLI 与脚本化入口 - 提升研发与运维效率。

### 低优先级（可以有）
1. 改写质量自动评分与对比 - 提升质量控制，但不阻塞主链路。
2. 智能风格推荐 - 改善体验，短期非必须。
3. 多平台发布（非微信） - 业务扩展项。

## AI增强功能

### AI增强 1：灵感评分与方向建议

**AI能力描述**：对采集内容进行打分与改写方向建议，帮助运营快速决策“是否采纳”。

**触发条件**：新灵感采集成功后自动触发。

**输入**：`title`、`content`、可选方向 `direction`。

**输出**：`score`、`reason`、`direction`、`insight`、`suggested_style`。

**示例**：
- 输入：AI Agent 行业动态文章
- 输出：`score=88`，建议风格 `tech_expert`，方向“聚焦落地案例与成本收益”。

---

### AI增强 2：上下文改写（相似度筛选参考）

**AI能力描述**：从同账户灵感库中筛选高相似参考文章，辅助改写输出更具一致性和可用性。

**触发条件**：改写请求 `use_references=true`。

**输入**：原文 `content`、风格 `style_preset`、灵感集合 `inspiration_records`、阈值 `similarity_threshold`。

**输出**：`content`（改写 HTML）、`used_references`、`reference_count`。

**示例**：
- 输入：SaaS 增长主题原文 + 10 条灵感
- 输出：选中 3 条高相似灵感，生成分析型改写稿。

---

### AI增强 3：定制改写指令注入（待补齐）

**AI能力描述**：把运营人员的“额外指令”直接注入系统提示，控制语气、结构、重点。

**触发条件**：改写请求中 `instructions` 非空。

**输入**：`instructions`，示例“增加 3 条可执行建议，减少术语”。

**输出**：符合定制要求的改写内容。

**示例**：
- 输入：`instructions=加入数据对比并压缩篇幅`
- 期望输出：含数据对比段落，篇幅控制在手机端可读长度。

---

### AI增强 4：自动模式选择（手动参考/自动参考/无参考）

**AI能力描述**：提供显式模式开关，避免当前“未手选参考即关闭引用”的隐式行为。

**触发条件**：用户在改写页选择模式。

**输入**：`rewrite_mode`（`manual`/`auto`/`none`）、可选 `inspiration_ids`。

**输出**：稳定、可解释的改写路径与日志标记。

**示例**：
- `auto`：系统自动选 3 条高相关灵感
- `manual`：严格按用户所选灵感改写
- `none`：纯原文改写，不引入参考

## 非功能性需求

### 性能要求
- `GET /api/health`：P95 < 100ms。
- `GET /api/stats`：P95 < 300ms（单机 SQLite，万级数据）。
- `POST /api/articles/{id}/rewrite`：单次超时上限 60s，超时必须返回可读错误。
- `POST /api/articles/{id}/publish`：单次超时上限 60s，失败必须可重试。

### 安全要求
- 凭证仅通过 `.env` 或账户配置管理，禁止写入代码仓库。
- 默认仅本机监听 `127.0.0.1`。
- API 错误信息不泄露密钥、Token、完整上游响应体。
- 账户敏感字段前端按需展示与脱敏。

### 可用性要求
- 关键动作（采集、改写、发布、广告位保存）必须有明确成功/失败反馈。
- 所有列表在空数据时提供空态，不出现白屏。
- 账户切换后 3 秒内完成主页面数据刷新。

### 兼容性要求
- 历史入口 `/admin` 必须兼容重定向到 `/`。
- 历史数据库需自动补齐新增字段（如 `rewrite_style`、`rewrite_references`）。
- 统计口径需兼容历史状态值，避免前端展示断裂。

## 技术栈建议

### 前端
- Vue 3 + Vue Router + Pinia：支持模块化后台和状态集中管理。
- Element Plus：快速构建运营后台组件。

### 后端
- Flask：轻量 API 层与静态托管。
- Pydantic：模型校验和配置管理。

### 数据库
- SQLite：单机部署成本低，适合中小规模内容运营。

### AI服务
- OpenAI 兼容接口（当前接入 Doubao Endpoint）：支持改写与评分统一调用。

- 广告位：发布到公众号正文头部或底部的固定图文内容块。
