# Frontend Standard: Vue Admin

## 目标

管理后台前端正式切换到 `Vue 3 + Vite` 作为后续标准架构。
这次调整的目标不是一次性重写全部页面，而是先建立统一的前端标准，让后续所有页面改版都基于同一套组件、设计令牌和 API 适配层继续开发。

## 当前约定

1. 后端继续使用 Flask，所有现有 `/api/*` 接口保持不变。
2. 新前端目录固定为 `frontend/admin`。
3. 新页面默认走 Vue 组件化开发，不再继续把复杂逻辑堆进单个 HTML 模板。
4. Vue 预览入口为 `/vue`（`/` 默认即 Vue）。

## 目录结构

```text
frontend/admin/
  package.json
  vite.config.js
  index.html
  src/
    App.vue
    components/
    lib/
    styles/
```

## 设计标准

1. 统一设计令牌：颜色、圆角、阴影、按钮、状态语义全部放在 `src/styles/main.css`。
2. 页面组件化：每个大页面拆成独立 Vue 组件，不再把页面逻辑写成一整段脚本。
3. API 统一封装：所有请求集中在 `src/lib/api.js`。
4. 状态语义统一：如写作链状态、失败状态、发布状态，统一放在 `src/lib/pipeline.js` 这类领域文件里维护。
5. 迁移顺序优先按业务价值推进：`概览 -> 写作管理链 -> 灵感库 -> 追踪中心 -> 账户号 -> 发布日志 -> 设置`。
6. 顶部工具栏和页面动作区优先采用单行布局，必要时允许横向滚动，不允许按钮自动堆叠成竖列。
7. 账户号页这类强表单页面，要显式区分：顶部页面动作、左侧目录、右侧表单、区块标题、字段帮助文案，避免把所有控制项挤进同一层级。

## 当前已完成

1. Vue 管理后台工程骨架已创建。
2. Vue 版本已完成基础壳层、全局导航、顶部工具栏。
3. `概览` 和 `写作管理链` 已提供 Vue 基线页面。
4. 其余页面统一按 Vue 组件标准持续迭代，不再保留旧版模板兜底入口。

## 启动方式

```bash
./run_admin.sh
```

启动脚本会自动：
1. 安装 Python 依赖
2. 安装 Vue 前端依赖
3. 构建 Vue 前端
4. 启动 Flask 管理后台

访问地址：
- 主入口：`/`
- Vue 预览：`/vue`

## 后续开发要求

1. 新设计稿优先落到 Vue 组件，而不是继续改旧版模板。
2. 任何新页面或大改版，先补组件边界，再落视觉，再接 API。
