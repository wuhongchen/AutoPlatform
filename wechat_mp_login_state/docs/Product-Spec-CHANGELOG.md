# Product Specification CHANGELOG

## [v2.6.1] - 2026-04-01

### Added
- 新增“独立功能菜单”要求：公众号关注与抓取子系统必须在后台侧边栏作为一级菜单展示，并与“灵感库”并列。
- 新增菜单产品命名：公众号关注与抓取子系统的正式菜单名为“公号雷达舱”。

### Modified
- 调整“功能 9”标题与描述：统一使用“公号雷达舱（公众号关注与抓取子系统）”命名，并补充菜单层级约束。
- 调整高优先级清单名称：将原“公众号关注与抓取子系统（多账户 + 后台/CLI 双入口）”更新为“公号雷达舱（公众号关注与抓取子系统）”。

## [v2.6.0] - 2026-04-01

### Added
- 新增“功能拆分边界”定义：将“灵感库中心能力”与“公众号关注与抓取子系统”明确拆分为两个独立模块，避免采集控制与运营入库耦合。
- 新增“公众号日批次投递”要求：公众号抓取子系统按账户每日生成抓取批次，并投递至灵感库中心能力统一入库。
- 新增“模块交接契约”要求：公众号抓取侧输出批次 ID 与来源字段，灵感库侧返回入库回执与映射结果，支持失败重放与链路追踪。

### Modified
- 重写“功能 2”定义：从“统一采集大杂烩”调整为“灵感库中心能力（入库编排与可运营视图）”，保留 URL/RSS/批次消费三类输入。
- 重写“功能 9”定义：从“抓取并直接入库”调整为“公众号关注与抓取子系统”，职责聚焦扫码登录、关注同步、文章列表拉取和日批次投递。
- 调整高优先级功能命名：同步更新优先级清单中的功能 2 和功能 9 名称，确保研发与运营对模块边界理解一致。

## [v2.5.0] - 2026-04-01

### Added
- 新增“多账户微信登录态采集控制台与 CLI 独立运行”功能定义：支持按账户独立扫码登录、关注列表同步、内容列表拉取、批量抓取并写入灵感库。
- 新增“后台 + CLI 双入口一致性”要求：后台动作与 CLI 命令必须复用同一套采集动作定义、状态持久化和日志结构。
- 新增“微信来源追踪字段”要求：灵感入库时补齐 `source_type=wechat_mp`、`source_account_id`、`source_mp_id`、`source_article_id`。

### Modified
- 强化“功能 2：灵感抓取与原文入库”定义：在 URL/RSS 基础上补充微信公众号登录态采集链路，并增加会话隔离、增量抓取、断点续抓和频控处理规则。
- 调整高优先级列表：将“多账户微信登录态采集控制台与 CLI 独立运行”列为当前版本高优先级能力。
- 扩展非功能需求与技术栈建议：明确微信登录态隔离约束、后台与 CLI 共享服务层、以及 wechat-demo-cli 桥接回退策略。

## [v2.4.0] - 2026-03-31

### Added
- 新增“灵感来源扩展”要求：灵感库采集入口支持手工 URL 直投与 RSS 订阅源两种模式。
- 新增“RSS 定时采集任务”要求：支持按账户配置采集间隔、每轮抓取上限、来源白名单，并输出任务执行统计与来源健康状态。
- 新增“采集去重与来源追踪”要求：入库时必须保存来源类型（manual_url/rss/feed_url）并执行标准化 URL 去重，保障灵感库质量。

### Modified
- 强化“功能 2：灵感抓取与原文入库”定义：从“URL 抓取”升级为“URL + RSS + 定时任务”的统一采集能力，并明确异常处理与质量回写规则。
- 调整功能优先级边界：将 RSS 能力从“低优先级扩展方向”提升为当前版本高优先级能力，低优先级仅保留非 RSS 外部来源扩展。
- 补充 AI 增强方向：新增基于订阅源质量自动推荐保留/降频/停用的能力定义。

## [v2.3.0] - 2026-03-31

### Added
- 新增“对话式运营助手”能力定义：支持通过类似 OpenClaw 机器人的对话入口查询近期灵感、近期改写、近期发布和近期失败记录，并返回结构化结果与下一步动作建议。
- 新增“近期内容固定动作”要求：对话入口可对最近内容执行送入写作链、标记跳过、重试改写、重试发布、打开原文、打开改后稿等标准动作。
- 新增“对话确认机制”要求：对账户级配置变更、发布动作、批量动作等高风险操作必须先返回预览与差异，再等待明确确认。

### Modified
- 强化“后台运行控制、对话式配置与定时调度”定义：明确对话入口是后台第二入口，不是替代品，且不得绕过人工审核、发布确认和高风险删除。
- 明确对话与后台的一致性要求：动作命名、账户上下文、参数校验、状态文案和写入结果必须统一复用，避免双轨体验和双事实源。
- 补强 OpenClaw 场景用户诉求：除了触发流程，还要支持通过对话配置系统参数和管理近期内容。

## [v2.2.0] - 2026-03-31

### Added
- 新增“写作管理链页面级展示规范”要求：状态看板必须包含节点 icon、数量、节点说明、任务卡动作、右侧详情联动和下一步操作提示。

### Modified
- 强化“改写流水线与状态机流转”定义：不再满足于工程状态罗列，明确要求写作管理链页面解决状态重叠、信息不直观、缺少 icon 语义和详情区不可用等问题。
- 明确看板布局规则：横向滚动只能发生在看板内部，不能压住右侧详情区；较窄宽度下必须自动退化为安全布局。

## [v2.1.0] - 2026-03-31

### Added
- 新增“本地账户数据库与基础设置”要求：本地数据库正式承担账户、配置、任务索引、链路映射和日志持久化职责，不再继续以 JSON 文件作为主配置中心。
- 新增“飞书多维表格字段治理与展示优化”要求：灵感库、流水线、发布记录表需要重构字段命名、字段分组、默认视图和摘要字段。
- 新增“链路追踪”要求：必须能把灵感记录、流水线记录、改后文档、发布记录、任务日志串联成统一追踪链路。

### Modified
- 明确数据边界：飞书继续作为正文与业务协作中心，本地数据库只存系统元数据，避免双主内容源冲突。
- 将“多个账户串号”“链路断裂”“失败排查困难”“OpenClaw 与本地双运行模式冲突”固化为当前版本必须解决的问题，而不是泛泛描述。
- 将失败修复能力升级为“失败修复 + 链路追踪 + 可观测性”一体化要求。

## [v2.0.0] - 2026-03-30

### Modified
- 将产品定位从“单账号自动采集改写发布工具”升级为“基于微信公众号发布能力的多账户内容运营系统”。
- 将飞书明确为事实内容中心，承载灵感库、发布流水线、发布记录和文档协作，不再把系统描述成自闭环脚本。
- 将后台能力纳入正式产品范围，新增“运行控制”与“系统基础设置”两类操作域。

### Added
- 多账户基础设置：每个账户独立配置公众号参数、飞书参数、多维表格、默认模型、默认角色和提示词方向。
- 灵感抓取与 AI 评分闭环：抓取、分析、原文文档生成、图片转存、入库回填统一归档。
- 流水线状态机规范：`🧲 待改写`、`✍️ 改写中`、`🧾 待审核`、`🚀 待发布`、`📤 发布中`、`✅ 已发布`、`❌ 改写失败`、`❌ 发布失败`。
- 失败修复与可观测性要求：失败原因回写、任务日志、账户维度追踪。
- AI 增强规划：账户方向提示词、公众号风格增强、改写去重、风险扫描与效果回流。

### Removed
- 移除旧文档中“项目主要是单篇公众号自动发布流水线”的简化定义。
- 移除对产品边界不清的描述，例如默认将系统理解为单账号脚本和单一路径生图工具。

## [v1.1.0] - 2026-03-03

### Added
- **内容灵感库 (Content Inspiration Library)**:
  - 独立于现有发布流水线的灵感采集与筛选模块。
  - 支持微信公众号 URL 互动数据抓取（阅读量/点赞量）。
  - 集成 AI 爆款评分与推荐理由生成。
  - 实现基于飞书表格状态流转的自动化流转逻辑。

### Modified
- **小龙虾智能内容库**: 降级为单纯的“发布流水线进程库”，仅接收灵感库筛选后的任务。
