## [1.1.0] - 2026-04-12

### 变更类型：修改

### 新增功能
- AI 改写场景矩阵：新增手动参考、自动参考、无参考、流水线批改四类场景定义。
- AI 输入输出契约：新增 `instructions`、`inspiration_ids`、`rewrite_mode` 的产品约束。

### 修改功能
- AI 改写：从“仅描述改写能力”升级为“可执行场景 + 可验收规则”。
- 流水线：从“触发即可”升级为“明确当前能力与待补齐能力边界”。
- 多账户：补充账户切换与页面联动的一致性约束。

### 删除功能
- 无。

### 功能调整
- 流水线任务可视化优先级调整：低 -> 中。
- 改写质量守卫优先级调整：低 -> 中。

### AI增强调整
- 新增“定制改写指令注入（待补齐）”条目，明确当前实现缺口与目标行为。
- 新增“自动模式选择（manual/auto/none）”条目，解决引用逻辑不透明问题。

### 非功能性需求调整
- 增加改写与发布超时上限（60s）与失败可重试要求。
- 增加统计口径兼容历史状态值的约束。

### 技术栈调整
- 无新增技术组件，保持 Vue + Flask + SQLite + OpenAI 兼容接口。

### 影响范围
- AI 改写页面：需要新增模式开关与更清晰的请求参数约束。
- 后端改写接口：需要补齐 `custom_instructions` 注入模型链路。
- 流水线与统计：后续需要引入任务记录与状态映射。

### 兼容性说明
- 向后兼容。
- 迁移指南：现有 API 可继续使用；新增模式字段可作为可选参数逐步上线。

## 版本号规范

遵循语义化版本规范：`MAJOR.MINOR.PATCH`

- **MAJOR（主版本号）**：不兼容的 API 修改或重大功能变更
- **MINOR（次版本号）**：向下兼容的功能性新增
- **PATCH（修订号）**：向下兼容的问题修正

示例：
- `1.0.0` - 初始版本
- `1.1.0` - 新增功能，向后兼容
- `1.1.1` - 修复 bug，向后兼容
- `2.0.0` - 重大变更，不向后兼容
